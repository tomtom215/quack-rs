# Your First Extension

This page walks through a complete, working DuckDB extension: `hello_ext`, which registers a
`word_count(VARCHAR) → BIGINT` aggregate function.

The full source lives in [`examples/hello-ext/src/lib.rs`](https://github.com/tomtom215/quack-rs/blob/main/examples/hello-ext/src/lib.rs).

---

## Overview

An extension has four parts:

1. **State struct** — holds data accumulated during aggregation
2. **Callbacks** — `update`, `combine`, `finalize`, `state_size`, `state_init`, `state_destroy`
3. **Registration** — wire the callbacks to DuckDB via `AggregateFunctionBuilder`
4. **Entry point** — DuckDB's initialization hook, generated by `entry_point!`

---

## 1. The state struct

```rust
#[derive(Default, Debug)]
struct WordCountState {
    count: i64,
}

impl AggregateState for WordCountState {}
```

`WordCountState` is a plain Rust struct with `Default`. The `AggregateState` impl is a
marker — it has no required methods. `quack_rs::aggregate::FfiState<T>` wraps it in a
heap-allocated `Box<T>` behind a raw pointer, managing the lifecycle for you.

---

## 2. The callbacks

### `state_size` and `state_init`

These delegate entirely to `FfiState`:

```rust
unsafe extern "C" fn state_size(_info: duckdb_function_info) -> idx_t {
    FfiState::<WordCountState>::size_callback(_info)
}

unsafe extern "C" fn state_init(info: duckdb_function_info, state: duckdb_aggregate_state) {
    unsafe { FfiState::<WordCountState>::init_callback(info, state) };
}
```

`FfiState::size_callback` returns `size_of::<*mut WordCountState>()` — the size of a
pointer, which DuckDB allocates per group. `init_callback` `Box::new(T::default())`s a fresh
state and writes its pointer into the DuckDB-allocated slot.

### `update`

Called once per input batch. Reads strings from the input vector, counts words, and
accumulates into the per-group state:

```rust
unsafe extern "C" fn update(
    _info: duckdb_function_info,
    input: duckdb_data_chunk,
    states: *mut duckdb_aggregate_state,
) {
    let reader = unsafe { VectorReader::new(input, 0) };
    let row_count = reader.row_count();

    for row in 0..row_count {
        if unsafe { !reader.is_valid(row) } {
            continue;   // skip NULLs
        }
        let s = unsafe { reader.read_str(row) };
        let words = s.split_whitespace().count() as i64;

        let state_ptr = unsafe { *states.add(row) };
        if let Some(st) = unsafe { FfiState::<WordCountState>::with_state_mut(state_ptr) } {
            st.count += words;
        }
    }
}
```

### `combine`

Called during parallel aggregation. Merges source states into target states:

```rust
unsafe extern "C" fn combine(
    _info: duckdb_function_info,
    source: *mut duckdb_aggregate_state,
    target: *mut duckdb_aggregate_state,
    count: idx_t,
) {
    for i in 0..count as usize {
        let src = unsafe { FfiState::<WordCountState>::with_state(*source.add(i)) };
        let tgt = unsafe { FfiState::<WordCountState>::with_state_mut(*target.add(i)) };
        if let (Some(s), Some(t)) = (src, tgt) {
            t.count += s.count;
        }
    }
}
```

> **Pitfall L1**: DuckDB creates fresh zero-initialized target states before calling `combine`.
> You must copy **all** fields — not just the accumulator. `WordCountState` only has one field
> so this is trivial here, but in more complex aggregates (e.g., those with a `window_size`
> config) you must also copy configuration fields. See [Pitfall L1](../reference/pitfalls.md#l1-combine-must-propagate-all-config-fields).

### `finalize`

Writes each group's final result into the output vector:

```rust
unsafe extern "C" fn finalize(
    _info: duckdb_function_info,
    source: *mut duckdb_aggregate_state,
    result: duckdb_vector,
    count: idx_t,
    offset: idx_t,
) {
    let mut writer = unsafe { VectorWriter::new(result) };

    for i in 0..count as usize {
        let state_ptr = unsafe { *source.add(i) };
        match unsafe { FfiState::<WordCountState>::with_state(state_ptr) } {
            Some(st) => unsafe { writer.write_i64(offset as usize + i, st.count) },
            None     => unsafe { writer.set_null(offset as usize + i) },
        }
    }
}
```

### `state_destroy`

Frees the `Box<WordCountState>` for each group. Delegates to `FfiState`:

```rust
unsafe extern "C" fn state_destroy(states: *mut duckdb_aggregate_state, count: idx_t) {
    unsafe { FfiState::<WordCountState>::destroy_callback(states, count) };
}
```

`destroy_callback` calls `Box::from_raw` and then nulls the pointer — preventing double-free
([Pitfall L2](../reference/pitfalls.md#l2-state-destroy-double-free)).

## 3. Registration

```rust
unsafe fn register_word_count(con: libduckdb_sys::duckdb_connection) -> Result<(), ExtensionError> {
    unsafe {
        AggregateFunctionBuilder::new("word_count")
            .param(TypeId::Varchar)
            .returns(TypeId::BigInt)
            .state_size(state_size)
            .init(state_init)
            .update(update)
            .combine(combine)
            .finalize(finalize)
            .destructor(state_destroy)
            .register(con)
    }
}
```

The builder sets every required DuckDB C API property and calls
`duckdb_register_aggregate_function`. If DuckDB reports an error, `register` returns `Err`.

---

## 4. Entry point

```rust
entry_point!(hello_ext, |con| register_word_count(con));
```

This macro generates the `hello_ext_init_c_api` function that DuckDB calls when loading the
extension. Internally it:
1. Calls `duckdb_rs_extension_api_init` to initialize the C API dispatch table
2. Opens a connection via `duckdb_connect`
3. Calls your registration closure
4. Disconnects and returns `true`/`false` to DuckDB

## Building and testing

```bash
cargo build --release --manifest-path examples/hello-ext/Cargo.toml

# Test in DuckDB:
duckdb -cmd "
  LOAD './examples/hello-ext/target/release/libhello_ext.so';
  SELECT word_count(txt) FROM (VALUES ('hello world'), ('one two three'), (NULL)) t(txt);
"

# │ word_count(txt)│

```

## Unit tests (no DuckDB required)

Test your state logic in isolation using `AggregateTestHarness`:

```rust
#[test]
fn word_count_combine() {
    let mut h1 = AggregateTestHarness::<WordCountState>::new();
    h1.update(|s| s.count += 2);  // "hello world"

    let mut h2 = AggregateTestHarness::<WordCountState>::new();
    h2.update(|s| s.count += 4);  // "one two three four"

    h2.combine(&h1, |src, tgt| tgt.count += src.count);

    assert_eq!(h2.finalize().count, 6);
}
```

See the [Testing Guide](../testing.md) for the full test strategy.
